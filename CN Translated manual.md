## NAME (项目名)
**vnstat** - 基于终端的网络流量监控器

## SYNOPSIS (语法)
```
vnstat 
[ -5bDedhlmqstvy? ]
[ --95th ]
[ --add ]
[ --alert output exit type condition limit unit ]
[ --begin 时间点 ]
[ --config 配置文件 ]
[ --days [ limit ] ]
[ --db file ]
[ --dbdir directory ]
[ --dbiflist [ mode ] ]
[ --debug ]
[ --end 时间点 ]
[ --fiveminutes [ limit ] ]
[ --help ]
[ -hg ]
[ --hours [ limit ] ]
[ --hoursgraph ]
[ -i 网卡名 ]
[ --iface 网卡名 ]
[ --iflist [ mode ] ]
[ --json [ mode ] [ limit ] ]
[ --limit 限制条件 ]
[ --live [ mode ] ]
[ --locale locale ]

```

## DESCRIPTION (描述)
`vnStat` 是一个基于终端的网络流量监控器。它会为选定的网卡接口记录每 5 分钟、每小时、每天、每月和每年的网络流量日志。
实际上，它**不是**数据包嗅探器。流量信息是从 `proc(5)` 或 `sys` 文件系统中读取的（这取决于系统的可用性），因此无论网络流量速率如何，它所占用的系统资源都非常少。这也使得在大多数系统上，`vnStat` 甚至可以在没有 root 权限的情况下使用。

功能被分为两个命令。`vnstat` 命令的目的是提供一个接口，用于查询存储在数据库中的流量信息；而守护进程 `vnstatd(8)` 则负责数据的检索、缓存和存储。尽管守护进程作为服务一直在后台运行，但实际上它大部分时间都在两次数据更新之间处于休眠状态。

## OPTIONS (选项)

*   **`--95th`**
    显示当月当前的 第95百分数（95% 峰值带宽）输出。  
    此输出使用当月 5 分钟刷新一次的数据来计算接收、发送和总流量的 95th 百分位数流量速率。此外，还会显示接收、发送和总流量的最小、平均和最大流量速率。此输出要求 `5MinuteHours` 配置选项的值至少为 744（一个月的小时数），以便存储所有必要的数据，否则无法实现 100% 的数据覆盖。
*   **`--add`**
    为使用 `-i` 或 `--iface` 选项指定的网卡创建数据库条目。  
    在此操作期间守护进程可以处于运行状态，并且如果启用了 `RescanDatabaseOnSave` 配置选项，它将在 `SaveInterval` 分钟内自动开始监控该网卡，而无需重启。否则，需要重启守护进程才能监控新添加的网卡。
*   **`--alert output exit type condition limit unit`**
    根据给定参数的值，在满足标准的情况时显示警报、使用不同的退出状态码，或兼有。
    *   `output` 参数接受 0 ~ 3 的数字，控制命令何时（或是否）产生输出。  
    '0' 从不产生输出，'1' 始终产生输出，'2' 仅在预估使用量超过 `limit` 时显示输出，'3' 仅在超过 `limit` 时显示输出。
    *   `exit` 参数接受 0 ~ 5 的数字，控制命令的退出状态码。  
    '0' 始终使用退出状态码 0，'1' 始终使用退出状态码 1，'2' 在预估使用量超过 `limit` 时使用退出状态码 1（否则为 0），'3' 在超过 `limit` 时使用退出状态码 1（否则为 0）。  
    '4' 和 '5' 分别是 '2' 和 '3' 的等效项，区别在于满足条件时使用退出状态码 2 而不是 1。这有助于区分警报和错误，因为错误始终使用退出状态码 1。
    *   `type` 参数定义将 `limit` 与哪种时间范围类型的使用量进行比较。可用选项：  
    `h`, `hour`, `hourly`,  
    `d`, `day`, `daily`,   
    `m`, `month`, `monthly`,   
    `y`, `year`, `yearly`,   
    `p`, `95`, `95%`。
    *   `condition` 参数：定义是将 `limit` 与接收 (rx)、发送 (tx)、总流量或者时以上三者的预估使用量进行比较。可用选项：'rx', 'tx', 'total', 'rx_estimate', 'tx_estimate', 'total_estimate'。95th 百分位数 `type` 不支持预估选项。
    *   `limit` 是一个大于零且没有小数的整数，它使用 `unit` 参数定义的单位来定义流量使用限制。
    *   `unit` 接受以下选项：`B`, `KiB`, `MiB`, `GiB`, `TiB`, `PiB`, `EiB`, `KB`, `MB`, `GB`, `TB`, `PB`, `EB`。
    *   对于 95th 百分位数 `type`，提供以下速率选项：`B/s`, `KiB/s`, `MiB/s`, `GiB/s`, `TiB/s`, `PiB/s`, `EiB/s`, `kB/s`, `MB/s`, `GB/s`, `TB/s`, `PB/s`, `EB/s`, `bit/s`, `Kibit/s`, `Mibit/s`, `Gibit/s`, `Tibit/s`, `Pibit/s`, `Eibit/s`, `kbit/s`, `Mbit/s`, `Gbit/s`, `Tbit/s`, `Pbit/s`, `Ebit/s`。
    *   *注：使用量必须**超过** `limit` 才会触发警报。正好等于 `limit` 的使用量不会触发警报。预估计算不局限于 `condition` 参数中的预估选项，也可以通过在 `output` 或 `exit` 参数中使用预估选项来实现。缺少或无效的参数或参数组合将导致显示 `--alert` 专属的帮助输出。*
*   **`-b, --begin date`**
    使用 `date` 定义的特定日期/时间开始列表输出，而不是根据要显示的条目数来选择起点。如果数据库中不存在该 `date`，则将使用最接近的较晚日期。`date` 支持以下格式：`YYYY-MM-DD HH:MM`、`YYYY-MM-DD` 和 `"today"`。此选项只能与 `--json`、`--xml` 和列表输出一起使用。
*   **`--config file`**
    使用 `file` 作为配置文件，而不是使用自动配置文件搜索功能。如果多次使用 `--config`，命令行中靠后的文件中的配置设置将覆盖靠前文件中的设置。
*   **`-d, --days [limit]`**
    按天显示过去几天的流量统计信息。除非另有配置或使用可选的 `limit` 参数，否则列表长度将限制为 30 个条目。如果 `limit` 设置为 0，将显示数据库中存储的所有条目。
*   **`--db file`**
    使用 `file` 作为数据库文件，而不是从配置文件中指定的目录或硬编码的默认目录（如果没有配置文件）中搜索数据库。此选项会覆盖 `--dbdir`。
*   **`--dbdir directory`**
    使用 `directory` 作为数据库目录，而不是使用配置文件中指定的目录或硬编码的默认目录。如果同时定义了 `--db`，则忽略此选项。
*   **`--dbiflist [mode]`**
    列出当前在数据库中的网卡。如果未定义 `mode` 或设置为 0，则输出将使用单行详细格式。如果 `mode` 设置为 1，则输出每行包含一个网卡；如果设置为 2，则仅将网卡数量显示为单个数字。另请参见 `--iflist`。
*   **`-D, --debug`**
    显示额外的调试输出。
*   **`-e, --end date`**
    使用 `date` 定义的特定日期/时间结束列表输出，而不是数据库中的最新日期/时间。如果数据库中不存在该 `date`，则将使用最接近的较早日期。`date` 支持以下格式：`YYYY-MM-DD HH:MM` 和 `YYYY-MM-DD`。此选项只能与 `--json`、`--xml` 和列表输出一起使用。在列表输出中，预估行将被替换为求和行，其值代表各列的总和。仅当输出包含多行数据时才显示求和行。即使定义的 `date` 与当前日期相同，这也适用。Top 列表（`--top`）也需要同时使用 `--begin` 选项。
*   **`-5, --fiveminutes [limit]`**
    以 5 分钟为刷新率显示过去几小时的流量统计信息。除非另有配置或使用可选的 `limit` 参数，否则列表长度将限制为 24 个条目。如果 `limit` 设置为 0，将显示数据库中存储的所有条目。
*   **`-h, --hours [limit]`**
    按小时显示流量统计信息。除非另有配置或使用可选的 `limit` 参数，否则列表长度将限制为 24 个条目。如果 `limit` 设置为 0，将显示数据库中存储的所有条目。
*   **`-hg, --hoursgraph`**
    使用条形图显示过去 24 小时的每小时流量统计信息，随后是代表数值数据的表格。
*   **`-i, --iface interface`**
    选择一个特定的 `interface` 并仅对其应用操作。对于数据库查询，可以使用 `interface1+interface2+...` 语法合并两个或多个网卡的信息。使用合并语法时，提供的所有网卡必须是唯一的，并且必须存在于数据库中。  
    可选地，根据 `InterfaceMatchMethod` 配置设置，可以使用先前通过 `--setalias` 设置的 `alias`（别名）来代替 `interface`。使用别名时不支持合并语法。`-i, --iface` 选项是可选的，即使不显式使用该选项，也可以将 `interface` 作为命令行参数用于选择使用的网卡。
*   **`--iflist [mode]`**
    列出当前可用的网卡。如果未定义 `mode` 或设置为 0，则输出将使用单行详细格式。如果 `mode` 设置为 1，则输出每行包含一个网卡；如果设置为 2，则仅将网卡数量显示为单个数字。另请参见 `--dbiflist`。
*   **`--json [mode] [limit]`**
    以 json 格式显示所选网卡或所有网卡的数据库内容。除非键名另有指示，否则输出中的所有流量值均以字节为单位。可以使用可选的 `mode` 参数将输出限制为仅选定的信息。默认情况下，除 95th 百分位数输出外，显示所有内容。
    *   将 `mode` 设置为 `'s'` 将输出包含 5分钟、小时、天、月和年分辨率数据的最后 2 个条目的摘要。
    *   `'f'` 仅输出 5分钟分辨率条目，`'h'` 小时，`'d'` 天，`'m'` 月，`'y'` 年，`'t'` 流量最高的日期（top days），`'p'` 95th 百分位数。
    *   或者，与 `mode` 结合使用可选的 `limit` 参数可以限制输出中的条目数。
    *   `--json` 选项可与 `-l, --live` 和 `-tr` 选项结合使用，此时 `mode` 或 `limit` 对输出没有任何影响。输出中的 `jsonversion` 字段包含 API 版本信息。仅当以前存在的内容的名称或结构发生更改时，它才会更改。相比之下，`vnstatversion` 字段仅作为额外信息存在。
*   **`--limit limit`**
    将列表输出中显示的最大条目数设置为 `limit`。使用 `--limit` 会覆盖默认的列表条目限制值以及直接为列表查询给出的可选 `limit` 参数。如果 `limit` 设置为 0，将显示数据库中存储的所有条目。`--limit` 也可用于控制 `--json` 和 `--xml` 输出的长度。
*   **`-l, --live [mode]`**
    实时显示所选网卡的当前传输速率，直到中断。如果运行时间超过 10 秒，中断后将显示统计信息。可以使用可选的 `mode` 参数在执行期间选择显示每秒数据包数（模式 0）或传输计数器（模式 1）。`--style` 也可用于影响输出的布局。如果与 `--json` 选项结合使用，输出将采用 json 格式。
*   **`--locale locale`**
    使用 `locale` 代替配置文件中指定的区域设置或系统默认设置（如果没有配置文件）。
*   **`--longhelp`**
    显示完整的选项列表。
*   **`--merge source destination`**
    将网卡数据从 `source`（源）数据库合并到 `destination`（目标）数据库。除非指定了网卡，否则 `source` 中的所有网卡都将合并到 `destination` 中。
    *   如果 `destination` 不存在，将创建一个新数据库。
    *   如果 `destination` 中不存在同名网卡，则将直接执行从 `source` 复制数据的操作。
    *   如果 `destination` 中已存在同名网卡，则将执行累加合并操作。
    *   `source` 数据库始终以只读方式访问，合并操作绝不会修改它。对 `destination` 数据库的更改无法撤消，因为不支持减法操作。
    *   执行合并需要使用附加参数进行确认。如果未提供此附加参数，则将显示操作预览和附加指导。
    *   `source` 和 `destination` 都可以直接指向数据库文件，或者使用替代的 `file:interface` 语法，其中数据库文件后缀为 `:`，后跟网卡名称。当用作 `source` 时，替代语法允许指定从 `source` 合并到 `destination` 的一个网卡，而不是合并所有网卡。此外，可以在 `destination` 中使用替代语法来指定数据将合并到哪个网卡。
*   **`-m, --months [limit]`**
    按月显示过去几个月的流量统计信息。除非另有配置或使用可选的 `limit` 参数，否则列表长度将限制为 12 个条目。如果 `limit` 设置为 0，将显示数据库中存储的所有条目。
*   **`--oneline [mode]`**
    使用单行可解析格式显示所选网卡的流量摘要。输出包含 15 个字段，使用 `;` 作为字段分隔符。
    1.  第 1 个字段包含输出的 API 版本信息（仅在字段内容或结构发生变化时更改）。
    2.  网卡名称
    3.  今天的时间戳
    4.  今天的接收量 (rx)
    5.  今天的发送量 (tx)
    6.  今天的总流量
    7.  今天的平均流量速率
    8.  当前月份的时间戳
    9.  当前月份的接收量
    10. 当前月份的发送量
    11. 当前月份的总流量
    12. 当前月份的平均流量速率
    13. 历史总接收量
    14. 历史总发送量
    15. 历史总流量
    *   可选的 `mode` 参数可用于强制所有字段以字节为单位输出，且不显示单位本身。
*   **`-q, --query [mode]`**
    强制数据库查询模式。可以使用可选的 `mode` 参数覆盖默认查询模式。当数据库中有多个网卡时，`'a'` 会使用简短摘要输出，否则使用常规摘要输出。无论数据库中有多少网卡，`'s'` 都会为一个接口显示常规摘要输出。如果未定义可选的 `mode` 参数，则将使用配置的 `QueryMode`。
*   **`--remove`**
    删除使用 `-i` 或 `--iface` 指定的网卡的数据库条目并停止监控它。在此操作期间守护进程可以处于运行状态，并将自动检测到更改。
*   **`--rename name`**
    将数据库中使用 `-i` 或 `--iface` 指定的网卡重命名为新名称 `name`。新名称不能已存在于数据库中。此操作不会导致任何数据丢失。在此操作期间守护进程 **不应** 处于运行状态。
*   **`-ru, --rateunit [mode]`**
    交换配置的速率单位。如果速率配置为以字节显示，则存在此选项时速率将以比特显示。同样，如果速率配置为以比特显示，则存在此选项时速率将以字节显示。或者，可以使用 0 或 1 作为此选项的 `mode` 参数，以便在字节 (0) 和比特 (1) 之间进行选择，而不管配置文件设置如何。
*   **`--setalias alias`**
    将 `alias` 设置为所选网卡的别名，以便在查询中显示。可以通过为 `alias` 指定空字符串来删除设置的别名。在此操作期间守护进程可以处于运行状态。
*   **`-s, --short`**
    使用简短输出模式。当数据库中有多个可用网卡且未选择特定网卡时，也会使用此模式。
*   **`--showconfig`**
    使用与配置文件本身相同的格式显示当前配置。
*   **`--style number`**
    修改输出的内容和样式。将 `number` 设置为：
    *   `0`：更窄的输出
    *   `1`：启用条形图列
    *   `2`：与上一项相同，但在摘要输出中显示平均流量速率
    *   `3`：在所有支持的输出中启用平均流量速率
    *   `4`：在 `-l, --live` 和 `-tr, --traffic` 模式下禁用终端控制字符的使用。
*   **`-t, --top [limit]`**
    显示历史流量最高的日期。除非另有配置或使用可选的 `limit` 参数，否则列表长度将限制为 10 个条目。如果 `limit` 设置为 0，将显示数据库中存储的所有条目。当与 `--begin` 以及可选的 `--end` 一起使用时，列表将使用每日数据生成，而不是单独的 top 条目。每日数据的可用性定义了特定日期查询可以访问的边界。
*   **`-tr, --traffic [time]`**
    计算在给定的 `time` 秒内通过所选网卡的流量。如果未指定数字参数，`time` 将为 5 秒。如果与 `--json` 选项结合使用，输出将采用 json 格式（但在这种情况下，不会显示结果前的倒计时）。`--style` 也可用于影响输出的布局。
*   **`-v, --version`**
    显示当前版本。
*   **`--xml [mode] [limit]`**
    以 xml 格式显示所选网卡或所有网卡的数据库内容。除非键名另有指示，否则输出中的所有流量值均以字节为单位。可以使用可选的 `mode` 参数将输出限制为仅选定的信息。默认情况下，除 95th 百分位数输出外，显示所有内容。
    *   将 `mode` 设置为 `'s'` 将输出包含 5分钟、小时、天、月和年分辨率数据的最后 2 个条目的摘要。
    *   `'f'` 仅输出 5分钟分辨率条目，`'h'` 小时，`'d'` 天，`'m'` 月，`'y'` 年，`'t'` 流量最高的日期，`'p'` 95th 百分位数。
    *   或者，与 `mode` 结合使用可选的 `limit` 参数可以限制输出中的条目数。输出中的 `xmlversion` 字段包含 API 版本信息。仅当以前存在的内容的名称或结构发生更改时，它才会更改。相比之下，`vnstatversion` 字段仅作为额外信息存在。
*   **`-y, --years [limit]`**
    按年显示过去几年的流量统计信息。除非另有配置或使用可选的 `limit` 参数，否则列表默认显示所有条目。如果 `limit` 设置为 0，也将显示数据库中存储的所有条目。
*   **`-?, --help`**
    显示命令选项摘要。

## FILES (文件)
*   `/var/lib/vnstat/`
    默认数据库目录。
*   `/etc/vnstat.conf`
    配置文件。如果存在 `$HOME/.vnstatrc`，则不使用此文件。更多信息请参见 `vnstat.conf(5)`。

## EXAMPLES (示例)
*   `vnstat`
    显示默认网卡或多个受监控网卡的流量摘要。
*   `vnstat -i eth0+eth1+eth3`
    显示 eth0、eth1 和 eth3 网卡合并后的流量摘要。
*   `vnstat -i eth2 --xml`
    以 xml 格式输出关于 eth2 网卡的所有信息。
*   `vnstat --json`
    以 json 格式输出所有受监控网卡的所有信息。
*   `vnstat -i eth0 --setalias local`
    为 eth0 网卡赋予别名 "local"。在之后查询 eth0 时，该信息将作为标签可见。
*   `vnstat -i eth2 --remove`
    删除 eth2 网卡的数据库条目并停止监控它。

## RESTRICTIONS (限制)
更新需要执行的频率必须至少与网卡产生足够流量以溢出内核网卡流量计数器的频率一样高。否则，可能会漏掉部分流量。对于 32 位网卡流量计数器，两次更新之间的最长时间取决于网卡传输 4 GiB 数据的速度。请注意，不能保证 64 位内核的所有网卡都具有 64 位网卡流量计数器。计算出的理论溢出时间（最长更新时间间隔）为：

| 接口带宽 | 理论溢出时间 |
| :--- | :--- |
| 10 Mbit | 54 分钟 |
| 100 Mbit | 5 分钟 |
| 1000 Mbit | 30 秒 |

无法监控虚拟网卡和别名网卡，因为内核不提供此类网卡的流量信息。此类接口通常命名为 `eth0:0`、`eth0:1`、`eth0:2` 等，其中 `eth0` 是被设置别名的实际接口。

使用**较长的日期输出格式**可能会导致显示的列不对齐，如果日期长度超过了固定大小的分配空间。
